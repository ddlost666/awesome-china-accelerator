## 一、流量方向完全相反

### 普通 VPN：从「国内」走向「海外」

```
[国内用户]  →  [VPN 加密通道]  →  [海外服务器]  →  [Google / Twitter / YouTube]
```

- 用户在国内
- 想访问海外服务（YouTube、Google 等）
- 通过 VPN 把流量送到海外节点
- 出口 IP 是海外 IP

### 回国加速器：从「海外」走向「国内」

```
[海外用户]  →  [加速器加密通道]  →  [国内节点]  →  [B站 / 王者荣耀 / 爱奇艺]
```

- 用户在海外
- 想访问国内服务（B 站、王者荣耀、爱奇艺等）
- 通过加速器把流量送到国内节点
- 出口 IP 是中国大陆 IP

**核心区别**：方向完全相反，**两者无法相互替代**。

## 二、技术架构不同

### 2.1 传统 VPN 的工作原理

传统 VPN 采用**通用代理协议**（OpenVPN / WireGuard / IPSec 等），目的是「打通一条加密隧道」，让用户能访问被本地网络封锁的内容。

特点：
- 通用协议，适用于多种场景
- 节点部署灵活
- 速度受协议开销影响
- 不针对特定应用做优化

### 2.2 回国加速器的工作原理

回国加速器采用**专线 + 应用层优化**架构：

- **国内多区域节点**：北京、上海、广州、香港等多地节点
- **专线传输**：与公共互联网分离，绕开跨境出口拥堵
- **应用层优化**：针对王者荣耀、爱奇艺、B 站等特定应用做路由优化
- **智能节点匹配**：根据用户所在地与目标应用自动选择最佳节点

## 四、为什么不能用普通 VPN 替代回国加速器？

### 4.1 出口 IP 不对

即使你用美国 VPN 连接到日本节点，**出口 IP 仍是日本（海外）IP**，B 站和爱奇艺照样会拦截。

普通 VPN 不会在国内部署节点，也不会针对王者荣耀、B 站做路由优化，延迟和速度都很差。

### 4.3 协议不适配

回国场景的核心挑战是**跨境出口拥堵**，普通 VPN 协议没有针对这一点做优化。
